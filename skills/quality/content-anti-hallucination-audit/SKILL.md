# 内容去AI幻觉审计

对AI生成的网站/文档进行系统性的逐条溯源验证，确保所有公开内容可被真实数据支持。

## 触发条件

- 用户要求"去AI幻觉"/"anti-hallucination"/"事实核查"
- AI生成的营销网站、产品文档、新闻稿上线前
- 发现AI编造的数字、名称、时间线等不实信息

## 审计流程

### Phase 1: 提取待验证主张

从目标内容中提取所有可验证的主张：
- 公司名称、地址、成立时间
- 产品名称、版本号、技术参数（精度/速度/产能）
- 合作伙伴、客户名称
- 数据指标（市占率/合作数量/专利数）
- 时间线事件（产品发布/获得认证/里程碑）
- 引用的法律法规、标准编号

### Phase 2: 多源交叉验证

对每条主张进行2+独立来源验证：

**外部公开源（优先）：**
```
web_search "公司全称 + 关键词"
web_search "品牌名 + 产品名"
web_search site:公司官网域名
browser_navigate 公司官网 url
```

**内部源文档：**
```
search_files path=档案目录 pattern=*关键词*
read_file 相关文档
```

### Phase 3: 三级分类

| 级别 | 定义 | 示例 |
|------|------|------|
| ✅ 可证实 | 2+独立来源一致 | "公司地址：天河区" — 官网+招聘网同时确认 |
| 🔴 需修正 | AI编造，与真实数据矛盾 | "5.0版本" — 官网无此版本号 |
| 🟡 待确认 | 仅1个来源或无公开来源 | "50+知识产权" — 未公开可验证数 |

### Phase 4: 修正策略

- 🔴需修正 → 替换为可证实数据
- 🟡待确认 → 删除或用可证实数据替代（如"100+合作单位"替代"50+知识产权"）
- 🟡待确认但可能正确 → 标注 `[待确认]` 保留
- ✅可证实 → 保留

### Phase 5: 部署后验证

```bash
# 验证幻觉词已清除
curl -sk https://example.com | grep -c "幻觉词"

# 验证可溯源数据存在
curl -sk https://example.com | grep -c "可证实数据"
```

## 常见AI幻觉陷阱

| 幻觉类型 | 典型表现 | 预防 |
|---------|---------|------|
| 版本号编造 | 系统名加"X.0" | 只写确认的版本号 |
| 地址编造 | 用城市+常见区名填空 | 查工商/官网/招聘 |
| 量化数据膨胀 | "50+"实则"未公开" | 找不到源就别写数字 |
| 时间线美化 | 把成立年份写晚 | 查新闻/工商注册 |
| 技术术语转化 | AGV→AMR用错 | 查官网产品名 |
| 客户名单编造 | 列出知名机构 | 只写已公开合作的 |

## 进阶：发现真实官网

当AI生成的内容可能不如真实官网具体时：

1. **搜索公司品牌名 + 已知产品名**，如 `web_search "明远生物" AMR 补苗机器人`
2. **尝试搜索已知域名**，如 `web_search site:your-domain.com`
3. **从搜索结果提取真实官网URL**，全量抓取产品页/关于页
4. **逐项对比AI内容与真实官网**：版本号/地址/产品名/合作数据

## 误报处理

`grep "5\\.0"` 可能匹配到CSS内联样式（如 `1500px`、`aspect-ratio: 4/3`）。
验证时需排除 `<style>` 标签内的CSS值，只检查HTML文本内容。
使用 `grep -oP '>([^<]*5\\.0[^<]*)<'` 定位文本中的实际匹配。

## 已知局限

- 搜索引擎结果可能过时或不完整
- 某些真实数据不存在于公开网络
- 用户提供的内部文档可能包含过时信息
- `[待确认]` 标注不能替代用户最终审批
- `grep` 全文匹配会误报CSS/JS中的数字字符串 → 用DOM文本提取替代
