## 一、构图模式详解

### hero_centric — 主体居中
```
┌─────────────────────┐
│                     │
│    [ 主标题 ]        │  ← 画面上方 1/4
│    [ 副标题 ]        │
│                     │
│      ┌──────┐       │
│      │ 主体  │       │  ← 画面中央 60%
│      └──────┘       │
│                     │
│          [CTA 按钮]  │  ← 右下角
└─────────────────────┘
```
- 适用: 社交 feed、产品展示、人物广告
- 主体占比: 50-65%
- 文字区域: 上方 25%

### split_screen — 左右分割
```
┌────────┬──────────┐
│  问题   │  方案    │
│  暗色   │  亮色    │
│  负面   │  正面    │
└────────┴──────────┘
```
- 适用: 竞品对比、前后对比
- 分割线: 正中或黄金比例 (38:62)
- 文字: 各占一侧

### text_overlay — 全幅背景 + 文字叠加
```
┌─────────────────────┐
│  [主标题 白色大字]    │
│  [副标题]            │
│                     │
│    全幅背景图         │
│    (产品/场景)        │
│                     │
│              [CTA]   │
└─────────────────────┘
```
- 适用: 情感型、品牌形象广告
- 文字需在背景上清晰可读（深色背景 + 白字，或加半透明遮罩）
- 背景图不可太复杂（否则文字不可读）

### grid_card — 卡片式布局
```
┌─────────────────────┐
│  ┌──────┐ ┌──────┐  │
│  │ 数据1 │ │ 数据2 │  │
│  └──────┘ └──────┘  │
│  ┌──────┐ ┌──────┐  │
│  │ 数据3 │ │ 数据4 │  │
│  └──────┘ └──────┘  │
│      [主标题] [CTA]  │
└─────────────────────┘
```
- 适用: B2B、数据密集型、LinkedIn
- 卡片间距: 16px
- 每卡片: 图标 + 数字 + 说明文字

### minimal_icon — 极简图标
```
┌─────────────────────┐
│                     │
│        ✦            │  ← 中心图标 40%
│                     │
│    [主标题]          │
│    [副标题]          │
│                     │
└─────────────────────┘
```
- 适用: 高端品牌、科技、奢侈品
- 大量留白 (≥ 50%)
- 字体纤细、高级感

### diagonal_dynamic — 对角线动感
```
┌─────────────────────┐
│  ╱                  │
│ ╱   [主标题]         │
│╱                    │
│  [主体元素]          │
│    ╲    [CTA]       │
│     ╲               │
└─────────────────────┘
```
- 适用: 运动、年轻化、抖音
- 对角线角度: 15-30°
- 色彩: 高饱和、对比强

## 二、色彩系统

### 色彩策略矩阵

| 策略 | 主色使用 | 辅色 | 强调色 | 背景 | 情绪 |
|------|---------|------|--------|------|------|
| brand_primary | 60% | 30% | 10% | 白/浅灰 | 专业、可信 |
| complementary | 40% | 20% | 40% | 深色 | 活力、冲击 |
| analogous | 50% | 30% | 20% | 浅色 | 和谐、柔和 |
| monochrome | 80% | 15% | 5% | 同色系深浅 | 高端、克制 |
| high_contrast | 30% | 10% | 60% | 黑/白 | 醒目、直接 |

### 色调情绪映射

| tone | 主色倾向 | 示例色 | 辅色 |
|------|---------|--------|------|
| professional | 蓝 | #0052D9 | #366EF4 |
| playful | 橙/黄 | #FF6B35 | #FFD700 |
| luxury | 黑/金 | #1A1A1A | #C9A96E |
| minimal | 灰/白 | #2C2C2C | #F5F5F5 |
| bold | 红 | #FF0000 | #FF4444 |
| warm | 暖橙 | #FF7B42 | #FFB088 |

## 三、字体系统

### 中文字体选择

| 字体 | 风格 | 适用 |
|------|------|------|
| 思源黑体 Bold | 现代、清晰、有力 | 主标题 (默认) |
| 思源黑体 Regular | 干净、中性 | 副标题、正文 |
| 思源黑体 Heavy | 极粗、冲击力 | bold 调性主标题 |
| 思源黑体 Light | 纤细、高端 | minimal/luxury 标题 |
| 思源宋体 Bold | 传统、庄重 | luxury、B2B 权威 |

### 字号层级规范

| 元素 | 最小 | 推荐 | 最大 |
|------|------|------|------|
| 主标题 | 36px | 48-56px | 72px |
| 副标题 | 20px | 24-28px | 32px |
| CTA 按钮文字 | 16px | 20-22px | 24px |
| 免责声明 | 10px | 12px | 12px |

### 字号比要求
- 主标题 : 副标题 ≥ 1.5:1
- 主标题 : CTA ≥ 2:1
- 副标题 : CTA ≥ 1.2:1

## 六、禁止反模式 (引用自 shared/anti_patterns.yaml)

以下反模式在 Stage 2 设计阶段就应避免（不等 Stage 4 审查发现）:

1. 多个概念的色彩策略完全相同 → 差异化强制要求
2. CTA 文字色与背景色 ΔE < 30 → 不可读
3. 主标题字号 < 36px → 任何平台都太小
4. 文字在复杂纹理背景上（无遮罩） → 必然不可读
5. 红绿配色 → 色盲不可区分
6. 灰色 (#CCCCCC 级别) 配白色背景 → 对比度不足
