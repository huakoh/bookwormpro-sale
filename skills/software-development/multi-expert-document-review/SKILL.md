# 多专家并行文档审查

> 用于创建和严格审查系统关键文档（soul.md / 宪法 / 安全策略等）的工作流。

## 触发条件

- 用户要求"多专家审查"、"严格审核"、"红队攻击"某个重要文档
- 创建新的系统灵魂/宪法/质量文档后需要验证
- 对现有文档做跨专家交叉审查发现盲区

## 工作流（4 阶段）

### 阶段 1：全量阅读源材料

读取所有相关源文件（宪法、反自负约束、系统操作指令等），不遗漏任何一章。源材料作为子代理审查的对照基准。

### 阶段 2：撰写初版

基于源材料浓缩为精华文档。目标：200-300 行，保持可读性和灵魂感，不是规则清单。

### 阶段 3：并行多专家审查（核心）

启动 3 路子代理并发审查，每路配备：
- 待审文档全文
- 对照源材料（与该专家视角相关的章节）
- 明确的审查标准和输出格式

**专家配置**：
| 路 | 专家 | 审查重点 | 输入源材料 |
|----|------|---------|-----------|
| 1 | 安全专家 | 安全红线/加密/认证/反腐败/NDA/Git安全 | 宪法第1/4/11/14/16章 |
| 2 | 质量诚信专家 | 反自负12条/交付标准/闭环度/DoD/跨AI一致性 | anti-arrogance.md + 宪法第2/9/13章 |
| 3 | 架构/红队专家 | 架构完整性 + 对抗性攻击向量（可合并或分两批） | 宪法第3/5/6/7/8/10/15章 + 装配索引 |

如 max_concurrent_children=3，分两批：先 1+2+3(架构)，再红队单独。

**子代理 prompt 要点**：
- 给出"遗漏项（源材料有目标文档无）/ 弱化项（表述程度不及源）/ 危险简化（概括语替代具体禁止）/ 修正建议（具体文字）"四维输出格式
- 红队专家额外要求：攻击向量列表 + 风险等级 + 利用方式 + 如果 AI 只记 soul.md 忘记宪法会在哪些场景犯错

### 阶段 4：聚合修正

1. 汇总三路审查结论，按 CRITICAL > HIGH > MEDIUM > LOW 排序
2. 修正所有 CRITICAL + HIGH 项
3. 特别关注：
   - 自我违规（soul.md 本身是否触犯了 NDA/安全条款）
   - 枚举→原则转换（列表式禁止改为原则性描述防绕过）
   - 退化安全网（为"仅剩此文件"场景增加不可违反底线）
4. 修正后写入

## 系统挂载

重要文档写完后必须确认引用链路：
1. 检查 CLAUDE.md 是否引用了该文件
2. 如未引用，在适当位置添加引用行
3. Windows 下 patcher 工具 CRLF 易失败 → 用 Python `write_file` 或 `content.replace()` 修改

## 陷阱

- **patch 工具 Windows CRLF 失败** — 已知问题，用 `python -c "content.replace()"` 替代
- **子代理找不到文件** — 子代理工作目录不同，必须给绝对路径
- **max_concurrent 限制** — 默认 3，4 路专家需分两批
- **soul.md 自身 NDA 违规** — 检查是否泄露了技能数量/Agent 数量等保密信息
- **话术黑名单自查** — soul.md 不应出现"核武级/世界级/生产级/100%准确率"等禁词（"生产级"修饰平台名除外但仍需警觉）
