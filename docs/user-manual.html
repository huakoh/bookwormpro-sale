BookwormPRO

自托管、自进化的 AI Agent — CLI · 消息机器人 · Web 控制台

BookwormPRO 是一个运行在你自己机器、VPS 或无服务器基础设施上的 AI 编程助手。它支持 30+ 大模型供应商、15+ 消息平台和 65+ 内置工具，通过闭环学习持续改进自身能力。

多模型路由

Anthropic、OpenAI、DeepSeek、Gemini、Qwen 等 30+ 供应商，自动故障转移

全平台交付

CLI、Telegram、Discord、Slack、微信、钉钉、飞书等 15+ 消息平台

工具 & 技能

65+ 内置工具 + 可扩展技能市场，从代码执行到浏览器控制

安全隔离

Docker / SSH / Modal / Daytona 等多种沙箱后端，命令审批机制

安装

从源码安装（当前方式）

git clone https://github.com/huakoh/BookwormPRO.git
cd BookwormPRO
pip install -e .
说明：BookwormPRO 尚未发布到 PyPI，当前需要 clone 仓库后本地安装。未来正式发布后可直接 pip install bookwormpro。

可选扩展

扩展功能安装命令
[web]Web 控制台pip install bookwormpro[web]
[voice]语音输入输出pip install bookwormpro[voice]
[messaging]全消息平台pip install bookwormpro[messaging]
[cron]定时调度pip install bookwormpro[cron]
[mcp]MCP 协议pip install bookwormpro[mcp]
[honcho]AI 记忆增强pip install bookwormpro[honcho]
提示：全量安装：pip install -e ".[web,voice,messaging,cron,mcp,honcho]"

系统要求

  • Python 3.11+
  • 内存：5GB（可调）
  • 模型要求最低 64K 上下文窗口

初始配置

首次运行 bookworm setup，交互式向导会引导你完成：

  1. 选择 AI 供应商并输入 API Key
  2. 选择默认模型
  3. 配置终端后端（本地 / Docker / SSH）
  4. （可选）配置消息平台
bookworm setup

配置完成后，所有设置存储在：

  • 主配置：~/.bookwormpro/config.yaml
  • 密钥文件：~/.bookwormpro/.env

第一次对话

bookworm
# 或显式进入对话模式
bookworm chat

进入交互界面后，直接输入自然语言即可。支持多行输入、历史记录和斜杠命令补全。

续接上次对话：bookworm -c 或 bookworm --continue

浏览器

通过 Chrome DevTools Protocol 连接真实浏览器，支持导航、截图、表单操作、JS 执行。

/browser connect           # 连接浏览器
/browser status            # 查看状态
/browser disconnect        # 断开

配置项

browser:
  inactivity_timeout: 120    # 空闲超时秒数
  command_timeout: 30        # 命令超时
  record_sessions: false     # 录制视频
  allow_private_urls: false  # 允许 localhost
  cdp_url: ""                # 持久化 CDP 端点

记忆 & 上下文

自动上下文压缩

当上下文使用率超过阈值时自动压缩，保留最近 N 条消息：

compression:
  enabled: true
  threshold: 0.50       # 50% 时触发
  target_ratio: 0.20    # 保留尾部 20%
  protect_last_n: 20    # 保护最近 20 条

配置详解

所有非密钥配置存储在 ~/.bookwormpro/config.yaml，密钥存储在 .env。

核心配置

# 默认模型
model: anthropic/claude-sonnet-4-6

# Agent 行为
max_turns: 90              # 最大对话轮次
api_max_retries: 3         # API 重试次数

# 提示缓存 (Anthropic)
prompt_caching:
  cache_ttl: "5m"          # 5m 或 1h

# 工具输出限制
tool_output:
  max_bytes: 50000
  max_lines: 2000
  max_line_length: 2000

修改配置

bookworm config set model deepseek/deepseek-v4-pro
bookworm config edit        # 在编辑器中打开
bookworm config wizard      # 重新向导

terminal:
  backend: docker
  docker_image: nikolaik/python-nodejs:python3.11-nodejs20
  docker_volumes:
    - "/home/user/workspace:/workspace"

定时任务

用自然语言创建定时任务，BookwormPRO 自动执行并通过消息平台推送结果。

/cron add "每天早上 9 点检查服务器状态并汇报"
/cron add "每周五下午 5 点生成周报"
/cron list
/cron pause <id>
/cron remove <id>
bookworm cron list
bookworm cron status

检查点 & 回滚

每次破坏性文件操作前自动创建检查点，支持一键回滚。

/rollback           # 查看可用检查点
/rollback 3         # 回滚到检查点 #3
/snapshot create    # 手动创建快照
/snapshot prune     # 清理旧快照
checkpoints:
  enabled: true
  max_snapshots: 50

语音模式

按 Ctrl+B 录音，自动转写为文字输入。回复可朗读。

/voice on     # 开启语音
/voice off    # 关闭
/voice tts    # 仅语音输出
/voice status # 查看状态

支持引擎：Edge TTS（免费）、ElevenLabs（高质量）。语音识别：Faster-Whisper（本地）。

多配置文件

为不同项目或身份创建隔离的配置环境：

bookworm --profile work     # 使用 "work" 配置
bookworm --profile personal # 使用 "personal" 配置

每个 Profile 有独立的 config.yaml、.env、会话历史和技能目录。

~/.bookwormpro/

├── checkpoints/         # 文件系统快照

常见问题

如何切换模型？

对话中输入 /model deepseek/deepseek-v4-pro，或命令行运行 bookworm model。

如何排查连接问题？

bookworm doctor

自动诊断 API 连通性、配置完整性、依赖状态。

如何从 OpenClaw 迁移？

bookworm claw migrate --dry-run  # 预览
bookworm claw migrate            # 执行迁移

API Key 放在哪里？

统一存放在 ~/.bookwormpro/.env，不要写入 config.yaml。

YOLO 模式安全吗？

YOLO 模式跳过所有危险命令审批。仅建议在 Docker 等隔离环境中使用。

如何限制 Token 消耗？

使用 /reasoning none 减少推理 Token，/fast 启用快速模式，或选择更小的模型。
