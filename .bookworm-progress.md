## 2026-04-26 03:04 · [P2 read_file CWD/管道漂移修复]

### Bug
P1 修复后暴露的独立问题: `bookworm chat -q "用 read_file 读 README.md"` 在任何场景 (含 stdout 重定向 / 工作目录绝对路径) 一律出现 `┊ 📖 读取文件 README.md 61.5s [error]`，agent 兜底回 "读取超时，文件可能不存在"。

### 真因 (3 个 Windows-only 缺陷叠加，与原假设 (a) cwd 被 hook 改写无关)

**B1 · 管道 drain 死锁** — `tools/environments/base.py:483` `_drain` 用 `select.select([fd], [], [], 0.1)` 监听子进程 stdout fd; Windows 上 `select` 只支持 socket，对管道 fd 立即抛 `OSError`，except 块吞掉异常 break 退出。结果输出全留在内核管道缓冲：小输出 (wc -c) bash 成功退出但 Python 读不到 → empty；大输出 (sed README 12KB) 管道填满 bash 阻塞写入 → 走完 60s `terminal.timeout` 假超时。

**B2 · CWD POSIX 形式回写** — `_update_cwd` 与 `_extract_cwd_from_output` 把 bash `pwd -P` 的 `/d/repos/hermes-agent` 直接灌回 `self.cwd`，后续 Popen 把它当 Windows 路径交给 `CreateProcess` 抛 `WinError 267 目录名称无效`。

**B3 · /tmp 双视图脱节** — `get_temp_dir` 只接受 POSIX 根 (`startswith("/")`) 否则回退 `/tmp`；Windows 下 git-bash 虚拟 `/tmp` Python 不可见，`open(self._cwd_file)` 直接 FNF，B2 的归一逻辑根本进不去。

| 文件:行 | 改动 | 作用 |

| `tools/environments/base.py:487-501` | `_drain` 在 `_IS_WINDOWS` 走阻塞 `os.read` 路径，依赖 bash 关闭 stdout 的 EOF 自然结束 | 修 B1 |
| `tools/environments/local.py:390-400` | `_update_cwd` 调换顺序：先 `_extract_cwd_from_output` 剥离 marker，再读文件归一覆盖 | 修 B2 |
